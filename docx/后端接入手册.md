## 一、快速开始

### 1.1 服务地址

| 环境 | HTTP Base URL | WebSocket URL |
|------|--------------|---------------|
| 本地开发 | `http://127.0.0.1:3001` | `ws://127.0.0.1:3001` |
| 公网部署 | `http://<服务器IP>:3001` | `ws://<服务器IP>:3001` |

前端通过环境变量配置地址：

```env
# client/.env
VITE_API_URL=http://127.0.0.1:3001
VITE_WS_URL=ws://127.0.0.1:3001
```

### 1.2 启动后端服务

```bash
cd 股票模拟器2.0/server
cp .env.example .env       # 配置环境变量
npm install
npm run dev                # 开发模式（热重载）
# 或
npm run build && npm start  # 生产模式
```

首次启动时自动：
- 创建 `data/stock_simulator.db` 数据库文件
- 建立所有数据表
- 导入 310 只股票种子数据
- 启动 5 秒价格更新引擎

### 1.3 完整接入流程示例（TypeScript）

```typescript
// 1. 注册账号
const registerRes = await fetch(`${API_URL}/api/auth/register`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ username: 'player1', password: '123456', displayName: '股神' })
})
const { data: { token } } = await registerRes.json()

// 2. 保存 token
localStorage.setItem('token', token)

// 3. 连接 WebSocket 接收实时行情
const ws = new WebSocket(`${WS_URL}/ws/market`)
ws.onmessage = (e) => {
  const msg = JSON.parse(e.data)
  if (msg.type === 'price_update') {
    // 更新股价 UI
    updatePrices(msg.data)
  }
}

// 4. 带 token 请求需要认证的接口
const buyRes = await fetch(`${API_URL}/api/trade/buy`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({ stockCode: 'SH600000', quantity: 100, orderMode: 'MARKET' })
})
```

## 二、基础约定

### 2.1 请求格式

所有请求均使用 JSON：

```
Content-Type: application/json
```

### 2.2 响应格式

**成功响应：**

```json
{
  "success": true,
  "data": { ... }
}
```

**失败响应：**

```json
{
  "success": false,
  "error": "错误描述（中文）",
  "code": "ERROR_CODE"
}
```

### 2.3 认证头

所有需要认证的接口，在 Header 中携带 JWT Token：

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### 2.4 API Key（远程访问）

**本地请求（127.0.0.1）**：无需 API Key，直接访问。

**远程访问（非 127.0.0.1）**：需要在请求头中携带 API Key：

```
X-API-Key: your-configured-api-key
```

如果服务端未配置 `API_KEY` 环境变量（留空），则拒绝所有远程访问。

### 2.5 数值规范

| 字段 | 类型 | 说明 |
|------|------|------|
| 资金、价格 | `number`（浮点数） | 保留 4 位小数 |
| 数量、股数 | `number`（整数） | 整数，最小单位 1 股 |
| 时间戳 | `number` | Unix 毫秒时间戳 |
| ID | `string` | UUID v4 格式 |

- 无自动刷新机制，过期后用户需重新登录

## 四、WebSocket 实时行情

### 4.1 连接

```
WebSocket: ws://127.0.0.1:3001/ws/market
```

> WebSocket 连接**不需要认证**，任何客户端均可订阅行情。

```typescript
const ws = new WebSocket('ws://127.0.0.1:3001/ws/market')

ws.onopen = () => console.log('行情连接成功')
ws.onclose = () => console.log('行情连接断开')
ws.onerror = (e) => console.error('行情连接错误', e)
ws.onmessage = (e) => {
  const msg = JSON.parse(e.data)
  handleMarketMessage(msg)
}
```

### 4.2 服务端推送消息格式

服务端每 **5 秒**推送一次所有股票的最新价格：

```json
{
  "type": "price_update",
  "data": [
    {
      "code": "SH600000",
      "name": "浦发银行",
      "price": 9.85,
      "changePercent": 1.34,
      "changeAmount": 0.13,
      "volume": 1234567,
      "turnover": 12170364.95,
      "timestamp": 1709337600000
    },
    {
      "code": "SZ000001",
      "name": "平安银行",
      "price": 10.25,
      "changePercent": -0.58,
      "changeAmount": -0.06,
      "volume": 987654,
      "turnover": 10123753.5,
      "timestamp": 1709337600000
    }
  ]
}
```

### 4.3 连接断线重连建议

```typescript
function connectMarketWS() {
  const ws = new WebSocket(WS_URL)

  ws.onclose = () => {
    // 5 秒后重连
    setTimeout(connectMarketWS, 5000)
  }

  ws.onmessage = (e) => {
    const msg = JSON.parse(e.data)
    if (msg.type === 'price_update') {
      store.updatePrices(msg.data)
    }
  }

  return ws
}
```

### 4.4 价格更新算法说明

服务端股价基于以下规则计算（每 5 秒一次）：

1. **随机游走**：每只股票独立随机波动，最大 ±2%
2. **AI 压力叠加**：前端 AI 引擎提交的净买卖量会影响股价方向
3. **涨跌停限制**：相对昨收价最大波动 ±10%，触及后停止变动

## 六、交易接口

> 所有交易接口均需 JWT 认证。服务端对每笔交易进行独立验证，不信任客户端传递的资金/持仓数据。

### 6.1 买入股票

```
POST /api/trade/buy
Authorization: Bearer <token>
```

**请求体：**

```json
{
  "stockCode": "SH600000",
  "quantity": 100,
  "orderMode": "MARKET"
}
```

**限价单请求体：**

```json
{
  "stockCode": "SH600000",
  "quantity": 100,
  "orderMode": "LIMIT",
  "price": 9.80
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| stockCode | string | 是 | 股票代码 |
| quantity | number | 是 | 买入数量（正整数） |
| orderMode | string | 是 | `MARKET`（市价）或 `LIMIT`（限价） |
| price | number | 限价时必填 | 限价委托价格 |

**成功响应（200）：**

```json
{
  "success": true,
  "data": {
    "order": {
      "id": "550e8400-...",
      "stockCode": "SH600000",
      "stockName": "浦发银行",
      "type": "BUY",
      "orderMode": "MARKET",
      "quantity": 100,
      "price": 9.85,
      "executedPrice": 9.85,
      "status": "EXECUTED",
      "fee": 2.955,
      "createdAt": 1709337600000,
      "executedAt": 1709337600000
    },
    "player": {
      "cash": 986651.045,
      "totalAssets": 1235552.045
    }
  }
}
```

> **注意**：市价单立即成交；限价单当前实现也立即按当前价成交（服务端未实现挂单队列）。

**交易费率：** 买入 0.03%，卖出 0.03%（按成交总额计算）

**失败响应：**


## 十三、AI 压力接口

### 13.1 提交 AI 价格压力

```
POST /api/market/ai-pressure
Authorization: Bearer <token>
```

此接口供前端 AI 引擎使用。前端 1000 个规则 AI 交易者产生买卖决策后，将净买卖量汇总提交给服务端，服务端在下次价格更新时将其叠加到股价计算中。

**请求体：**

```json
{
  "pressures": [
    { "code": "SH600000", "netVolume": 5200 },
    { "code": "SZ000001", "netVolume": -3100 },
    { "code": "SH601318", "netVolume": 8700 }
  ]
}
```

| 字段 | 说明 |
|------|------|
| code | 股票代码 |
| netVolume | 净买入量（正数=净买入，负数=净卖出） |

**成功响应：**

```json
{
  "success": true,
  "data": { "message": "AI压力已更新" }
}
```

> **压力影响公式**：`priceEffect = netVolume / 10000 * 0.01`，即每 1 万股净买入影响 0.01% 的价格变动。每轮价格更新后压力自动清零。

## 十五、已知问题与注意事项

以下问题已在代码审查中确认，前端接入时需特别注意：

### ⚠️ 问题 1：数据库引擎选型（重要）

**现状**：后端使用 `sql.js`（SQLite WASM 版本，原本为浏览器端设计）运行在 Node.js 服务端。`package.json` 中未包含 `better-sqlite3`，数据库采用内存 + 定期写文件的方式持久化。

**影响**：
- 服务端异常崩溃时，若最后一次 `saveDatabase()` 未执行，可能丢失部分数据
- 相比 `better-sqlite3`，性能略低，但对当前规模影响不大

**前端应对建议**：关键操作（买入、卖出、借款）后，检查响应数据并在本地同步更新，不要完全依赖下一次轮询。

---

### ⚠️ 问题 2：交易流程缺少数据库事务

**现状**：`TradeService.ts` 中的买入/卖出流程包含多个数据库写入步骤（创建订单 → 更新持仓 → 扣除资金 → 创建记录），但这些步骤未包裹在数据库事务中。

**影响**：极低概率下（服务器在写入过程中崩溃），可能出现资金已扣除但持仓未更新的情况。

**前端应对建议**：交易响应中包含最新的 `player`（资金）数据，以响应数据为准更新前端状态，不要依赖本地计算。

---

### ⚠️ 问题 3：限价单当前立即成交

**现状**：`TradeService.buy()` 和 `TradeService.sell()` 无论 `orderMode` 为 `MARKET` 还是 `LIMIT`，均以当前市价立即成交。服务端未实现挂单等待机制（撮合引擎）。

**影响**：限价单实际行为与市价单相同，委托价格（`price`）字段被忽略。

**前端应对建议**：目前可以正常展示限价单 UI，但需告知用户限价单当前以市价成交。待后续版本服务端实现撮合引擎后，此行为会改变。

---

### ⚠️ 问题 4：T+1 可卖数量由前端管理

**现状**：买入后 `availableQuantity`（可卖数量）记录在数据库为 0（T+1 市场）。T+1 解冻逻辑（即下一交易日变为可卖）当前需要前端通过 `PUT /api/player/settings` 推进 `tradingDay` 来触发，服务端不会自动执行。

**建议**：前端在推进交易日时，同时重新拉取 `/api/positions` 获取最新持仓状态，以服务端数据为准。

---

### ⚠️ 问题 5：`migrate.ts` 迁移脚本为占位实现

**现状**：`src/db/migrate.ts` 的 `runMigrations()` 函数为简化实现（打印日志后直接返回），实际建表逻辑在 `src/db/index.ts` 的 `initDatabase()` 中完成。

**影响**：`npm run migrate` 命令不执行任何实际操作。建表迁移在服务启动时自动完成，无需手动执行。

---

### ℹ️ 信息：WebSocket 连接管理

WebSocket 服务通过 HTTP Upgrade 机制实现，连接路径为 `/ws/market`。服务端不对 WebSocket 连接进行认证，所有客户端均可连接接收行情。如需限制访问，需在服务端 `app.ts` 中手动添加鉴权逻辑。

---

### ℹ️ 信息：成就检查时机

成就检查（`AchievementService.checkAchievements()`）目前需要在各个业务操作后**手动调用**。当前服务端在交易、借贷操作的 service 层并未主动触发成就检查——成就解锁可能依赖前端轮询 `/api/achievements` 接口。建议前端在每次交易后主动刷新成就列表。

## 附录 B：前端 API 客户端封装建议

```typescript
// client/src/api/client.ts
import axios from 'axios'

const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  timeout: 10000,
})

// 自动附加 token
apiClient.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 统一处理 401
apiClient.interceptors.response.use(
  (res) => res.data,
  (err) => {
    if (err.response?.status === 401) {
      localStorage.removeItem('token')
      window.location.href = '/login'
    }
    return Promise.reject(err.response?.data || err)
  }
)

export default apiClient
```
